❓ How it works

Every contract in this dashboard is treated as a transparent upgradeable proxy (EIP-1967). The proxy itself exposes only a fallback and a receive function; all other calls are forwarded by the proxy to the implementation contract.

When you click Withdraw this interface encodes the selected function selector and sends it as the data payload of an ETH transaction addressed to the proxy. The proxy's fallback delegates the call to the implementation, which executes the withdrawal logic and transfers ETH (or tokens) to you.

The ABI embedded in this dashboard matches exactly the proxy ABI you provided (AdminChanged, BeaconUpgraded, Upgraded events + payable fallback/receive).
