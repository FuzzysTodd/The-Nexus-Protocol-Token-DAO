Workflow Notes

This page is intentionally simple: save a target, run a reachability test, then export an audit trail. It is designed to stay readable and easy to verify during approvals.

API Paths

GET /healthok

Health probe used by the service test flow.

POST /api/service-urlsave

Stores the current service target and records the change.

POST /api/test-servicetest

Performs a server-side fetch against the saved URL.

GET /api/export-auditexport

Downloads the latest audit log as markdown or JSON.
